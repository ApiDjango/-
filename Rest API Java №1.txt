Задача: Разработать REST API на Java для управления задачами.

Требования к API:

    API должно предоставлять возможность создания, чтения, обновления и удаления задач.
    Каждая задача должна содержать следующие поля: идентификатор, название, описание, дата создания и статус (например, "в работе", "завершена", "отменена" и т.д.).
    API должно быть защищено авторизацией и аутентификацией пользователей.
    API должно использовать формат передачи данных JSON.
    API должно поддерживать CRUD-операции (создание, чтение, обновление и удаление) задач.
    API должно предоставлять возможность фильтрации и сортировки задач по различным полям, например, по статусу или дате создания.
    API должно обрабатывать ошибки и возвращать соответствующие коды ошибок и сообщения.
    API должно быть документировано.

Технические требования:

    Язык программирования: Java 8+.
    Фреймворк: Spring Boot.
    СУБД: PostgreSQL.
    Все зависимости должны быть управляемы с помощью Maven или Gradle.
    Для тестирования API необходимо использовать JUnit.
    Для документирования API использовать Swagger или OpenAPI.

Результаты проекта:

    Исходный код приложения на GitHub.
    Документация API в формате Swagger или OpenAPI.
    Примеры запросов и ответов на основные API-методы.
    Все зависимости должны быть управляемы с помощью Maven или Gradle.
    Возможность быстрого и простого развертывания API на локальном сервере с помощью Docker или других средств контейнеризации.
