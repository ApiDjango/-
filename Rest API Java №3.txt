Задача: Разработать REST API на Java для социальной сети.

Требования к API:

    API должно предоставлять возможность создания, чтения, обновления и удаления пользователей.
    Каждый пользователь должен содержать следующие поля: идентификатор, имя, фамилию, электронную почту, пароль, дату рождения, фото профиля и список друзей.
    API должно предоставлять возможность создания, чтения, обновления и удаления сообщений.
    Каждое сообщение должно содержать следующие поля: идентификатор, автора сообщения, текст сообщения, дату создания и список получателей.
    API должно быть защищено авторизацией и аутентификацией пользователей.
    API должно использовать формат передачи данных JSON.
    API должно поддерживать CRUD-операции (создание, чтение, обновление и удаление) пользователей и сообщений.
    API должно предоставлять возможность фильтрации и сортировки пользователей и сообщений по различным полям, например, по имени, 
		дате рождения или дате создания сообщения.
    API должно обрабатывать ошибки и возвращать соответствующие коды ошибок и сообщения.
    API должно быть документировано.

Технические требования:

    Язык программирования: Java 8+.
    Фреймворк: Spring Boot.
    СУБД: PostgreSQL.
    Все зависимости должны быть управляемы с помощью Maven или Gradle.
    Для тестирования API необходимо использовать JUnit.
    Для документирования API использовать Swagger или OpenAPI.

Результаты проекта:

    Исходный код приложения на GitHub.
    Документация API в формате Swagger или OpenAPI.
    Примеры запросов и ответов на основные API-методы.
    Все зависимости должны быть управляемы с помощью Maven или Gradle.
    Возможность быстрого и простого развертывания API на локальном сервере с помощью Docker или других средств контейнеризации.
